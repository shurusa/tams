# طمس (TAMS)

أداة تعمل بالكامل داخل المتصفح لإخفاء البيانات الحساسة من أي نص أو ملف قبل إرساله لنموذج ذكاء اصطناعي، مع إمكانية استرجاع البيانات الأصلية بدقة كاملة بعد ذلك.

تطوير **SHURU** — [shuru.sa](https://shuru.sa) · info@shuru.sa

---

## الفكرة

1. تضيف نصًا أو ملفًا يحتوي على بيانات حساسة.
2. تستبدل الأداة كل بيانات حساسة برمز مؤقت فريد، مثل `⟦EMAIL_1⟧`.
3. ترسل النص المُخفى لأي نموذج ذكاء اصطناعي تريده.
4. تلصق رد النموذج، وتستخدم جدول الربط (Mapping) لاسترجاع البيانات الأصلية مكان كل رمز.

كل المعالجة تتم محليًا داخل المتصفح — لا يُرسل أي نص أو جدول ربط لأي خادم خارجي.

## المزايا

- **إدخال متعدد المصادر**: لصق نص مباشر، أو رفع ملفات `txt` / `csv` / `docx` / `pdf`.
- **كشف تلقائي** للأنماط الشائعة:
  - البريد الإلكتروني
  - أرقام الجوال (سعودية/خليجية ودولية)
  - رقم الهوية/الإقامة (10 خانات تبدأ بـ 1 أو 2)
  - آيبان (IBAN)
  - بطاقات بنكية (فحص Luhn لتقليل الإيجابيات الكاذبة)
  - أرقام حسابات عامة (9–18 خانة)
- **مصطلحات مخصصة**: إضافة أي اسم أو كلمة حرة يدويًا لإخفائها (الأسماء الحرة يصعب كشفها تلقائيًا بدقة).
- **جدول ربط قابل للتنزيل** بصيغة JSON لاسترجاع البيانات لاحقًا.
- **استرجاع فوري**: لصق رد النموذج + جدول الربط → نص أصلي كامل جاهز للنسخ.
- خصوصية بالتصميم: لا حفظ تلقائي، لا اتصال بخادم، لا تتبع.

## طريقة الاستخدام

افتح ملف `tams.html` مباشرة في أي متصفح حديث (Chrome, Edge, Safari, Firefox). لا حاجة لتثبيت أو خادم.

### إخفاء البيانات
1. الصق النص أو ارفع ملفًا في تبويب **إخفاء البيانات**.
2. (اختياري) أضف مصطلحات مخصصة في حقل "مصطلحات إضافية للإخفاء".
3. اضغط **إخفاء البيانات الحساسة**.
4. انسخ النص المُخفى واستخدمه مع أي نموذج ذكاء اصطناعي.
5. نزّل **جدول الربط (JSON)** واحفظه في مكان آمن.

### استرجاع النص الأصلي
1. افتح تبويب **استرجاع النص الأصلي**.
2. الصق رد النموذج.
3. ارفع ملف جدول الربط أو الصق محتواه.
4. اضغط **استرجاع النص الأصلي** للحصول على النص الكامل.

## التقنيات المستخدمة

| الغرض | المكتبة |
|---|---|
| قراءة ملفات Word (`.docx`) | [mammoth.js](https://github.com/mwilliamson/mammoth.js) |
| قراءة ملفات PDF | [pdf.js](https://mozilla.github.io/pdf.js/) |
| الخطوط | IBM Plex Sans Arabic / IBM Plex Mono (Google Fonts) |

لا يوجد اعتماد على أي إطار عمل (framework) — HTML/CSS/JS خام في ملف واحد.

## هيكل الرموز (Tokens)

| النوع | نمط الرمز |
|---|---|
| بريد إلكتروني | `EMAIL_1`, `EMAIL_2`, ... |
| جوال | `PHONE_1`, `PHONE_2`, ... |
| هوية/إقامة | `NATIONAL_ID_1`, ... |
| آيبان | `IBAN_1`, ... |
| بطاقة بنكية | `CARD_1`, ... |
| رقم حساب عام | `ACCOUNT_NO_1`, ... |
| مصطلح مخصص | `CUSTOM_1`, ... |

كل رمز يُغلَّف بين ⟦ ⟧ لتقليل احتمال تعارضه مع محتوى النص الأصلي.

## القيود المعروفة

- كشف الأسماء الحرة وأسماء الشركات غير مضمون بدقة كاملة تلقائيًا — استخدم حقل "مصطلحات إضافية" لتغطيتها يدويًا.
- الأداة أسلوب إخفاء (tokenization)، وليست تشفيرًا معتمدًا؛ لا تُستخدم وحدها كحل امتثال لبيانات تخضع لمتطلبات نظامية صارمة.
- جدول الربط لا يُحفظ تلقائيًا في أي مكان بعد إغلاق الصفحة — فقدانه يعني عدم إمكانية الاسترجاع.
- قراءة ملفات docx/pdf تعتمد على تحميل مكتبات خارجية من CDN، لذا تتطلب اتصالًا بالإنترنت عند رفع هذه الصيغ تحديدًا (اللصق المباشر والملفات النصية يعملان دون اتصال).

## الخصوصية والأمان

- لا يوجد خادم خلفي (backend)؛ الأداة ملف HTML مستقل.
- لا تُستخدم `localStorage` ولا أي تخزين متصفح دائم.
- جدول الربط يُنزَّل كملف على جهاز المستخدم فقط، ولا يُرفع لأي مكان.
